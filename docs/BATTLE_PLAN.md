## 一、诚实诊断：当前系统离实战有多远

| 能力 | 当前状态 | 实战差距 | 严重度 |
|---|---|---|---|
| 选股 | 能跑，但用Mock数据或100只样本 | 需要全市场4000+只实时筛选 | 🔴 P0 |
| 回测验证 | 框架通了，用的是随机Mock数据 | 没有用真实历史数据验证过策略 | 🔴 P0 |
| 数据获取 | 逐只财务API，100只=5分钟 | 全市场需3-5小时，不可用 | 🔴 P0 |

| 可视化 | 静态HTML日报 | 够用，可后期增强交互 | 🟢 P2 |

## 二、P0：不解决无法实战（本周）

### P0-1: 用现有量化平台做真实回测

**为什么不自己写回测了？** 因为数据获取是瓶颈。聚宽/BigQuant 这些平台自带全市场财务数据，回测秒级完成。

**方案**：在聚宽（JoinQuant）上复现书中策略，用2021-2025全市场数据验证

```
操作流程：
1. 注册 joinquant.com（免费）
2. 把 strategies.yaml 中5条策略翻译为聚宽API代码
3. 跑2021-01-01 ~ 2025-12-31 全市场回测
4. 对比书中年化 vs 近5年实际表现
5. 淘汰失效策略，保留有效策略
```

**预期收获**：知道哪条策略还在赚钱、哪条已经失效。

### P0-2: 自建全市场快速数据管道

**问题根因**：AkShare 逐只财务API太慢。
**解决方案**：两个互补方案

| 方案 | 数据源 | 速度 | 覆盖 | 成本 |
|---|---|---|---|---|
| A | tushare pro | 全量秒级 | PE/PB/ROE/毛利率等 | 需积分(可免费获取) |
| B | baostock | 全量秒级 | 基础财务 | 完全免费 |
| C | 东方财富/新浪批量接口 | 全量秒级 | PE/PB/市值 | 免费但不稳定 |

**建议**：优先 tushare + baostock 双源互备。tushare 注册后获取积分（通过贡献代码/文档可得），基础数据免费。

### P0-3: 真实选股 + 真实收益联动

```
每天收盘后：
  AkShare/Tushare 拉全量PE/PB/ROE/PS/波动率
  → 因子计算（23个因子，全市场）
  → 打分排序 → TOP 20
  → 记录持仓成本价
  → 下一个交易日：对比收盘价 → 真实日收益
  → 不再用 random walk 假数据
```

## 三、P1：实战必需（本月）

### P1-1: 调仓信号系统

```
输入: 今日选股TOP20, 昨日持仓
输出: 
  买入: 新进入TOP20的股票 × N只
  卖出: 跌出TOP20的股票 × M只
  仓位: 每只等权, 总仓位不超过80%
  止损: 单只亏损>8%强制平仓
  止盈: 单只盈利>30%减半仓
```

### P1-2: 模拟盘对接真实账户跟踪

```
券商账户(只看不下单)
  → MiniQMT/PTrade 获取持仓+市值
  → tracker 记录每日净值
  → 与策略选股对比
  → 评估报告
```

### P1-3: 风险控制模块

```
- 市场情绪过热信号(涨停家数>200) → 减仓至50%
- 单日大盘跌幅>3% → 当日不买
- 连续3日跑输基准 → 暂停新开仓
- 最大回撤>15% → 全面暂停, 复盘策略
```

## 五、技术选型决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 回测平台 | 聚宽（JoinQuant）| 免费、全市场数据、社区资源最丰富 |
| 本地数据 | tushare + baostock | 互补覆盖、免费可用 |
| 实盘接口 | 待定（需券商开户后确认） | QMT自由度最高, PTrade更省心 |
| 选股引擎 | 保留现有 quant-system | 架构好、可扩展、GitHub版本控制 |
